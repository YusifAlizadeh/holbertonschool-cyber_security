# 🦈 Wireshark & Network Traffic Analysis

Welcome to the **Wireshark Basics** directory! This folder contains a collection of advanced display filters designed to identify and isolate specific network scanning patterns generated by tools like `nmap` and `arp-scan`.

## 📌 Overview

Understanding how scanning tools interact with the network is key to defense. This project focuses on **Traffic Analysis**: the ability to look at "raw" packets and identify the digital signature of various reconnaissance methods. Each file here contains a precise Wireshark filter string used to detect a specific type of scan.

## 🗂️ Traffic Detection Filters

| File Name | Nmap Command Detected | Detection Logic / Protocol |
| :--- | :--- | :--- |
| **`0-ip_scan.txt`** | `nmap -sO` | Detects IP Protocol scanning (identifying supported protocols). |
| **`1-tcp_syn.txt`** | `nmap -sS` | Filters for Stealth (Half-Open) scans using the SYN flag. |
| **`2-tcp_connect_scan.txt`** | `nmap -sT` | Detects full TCP 3-way handshake connection attempts. |
| **`3-tcp_fin.txt`** | `nmap -sF` | Identifies FIN scans (packets with only the FIN bit set). |
| **`4-tcp_ping_sweep.txt`** | `nmap -sn -PS/-PA` | Detects host discovery via TCP SYN or ACK pings. |
| **`5-udp_port_scan.sh`** | `nmap -sU` | Isolates UDP traffic targeting various ports for enumeration. |
| **`6-udp_ping_sweep.txt`** | `nmap -sn -PU` | Identifies network-wide host discovery using UDP probes. |
| **`7-icmp_ping_sweep.txt`** | `nmap -sn -PE` | Filters for standard ICMP Echo Request (Ping) sweeps. |
| **`8-arp_scanning.txt`** | `arp-scan -l` | Detects Layer 2 ARP broadcasts used for local host discovery. |

## 🛠️ Security Concepts Applied

* **Packet Inspection:** Analyzing individual TCP/IP headers to identify anomalies (like a FIN packet without a prior handshake).
* **Signature Recognition:** Learning that every scanning tool leaves a unique "fingerprint" in the traffic.
* **Incident Response:** Using Wireshark to investigate potential reconnaissance phases before an actual breach occurs.
* **Display Filters:** Mastering the syntax of Wireshark (e.g., `tcp.flags.syn == 1 && tcp.flags.ack == 0`) to cut through the noise of millions of packets.

## 🚀 How to use these filters
To use any of the filters in this directory:
1. Open a `.pcap` or `.pcapng` file in Wireshark (or start a live capture).
2. Copy the string from the corresponding `.txt` file.
3. Paste it into the **Display Filter bar** at the top and press Enter.
